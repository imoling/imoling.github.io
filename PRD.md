# iML Agent Desktop 产品介绍页 PRD

## 1. 产品定位与核心价值
iML Agent Desktop 是一个**“人人皆可定义的桌面级智能体”**。
- **定位**：基于第一性原理设计的、拥有本地操作权限的全能执行助手。
- **核心价值**：打破 AI 与操作系统的边界。iML Agent 不仅能“听”，更能通过**原生钩子（Native Hooks）**感应系统事件并直接接管桌面软件的操作流。

## 2. 目标受众
- **开发者/技术极客**：需要自动化处理复杂脚本、文件管理和环境配置。
- **生产力发烧友**：追求极致交互体验，希望 AI 能深度接入其日常办公流。
- **完美主义者**：对软件审美和隐私有极高要求。

## 3. 设计语言：Executive High-End (执行官级高端感)
- **色调**：深空黑 (#050505) 为主色，辅以品牌紫 (#8B5CF6) 的点状渗透。
- **质感**：磨砂玻璃、超细分割线、流光边框。
- **动态**：摒弃花哨动效，采用“响应式动力学” (Responsive Kinetics)——当鼠标悬停时提供物理反馈（如位移、光影变化）。

## 4. 页面结构 (模块化设计)

### 模块 A：Hero - 核心宣言
- **视觉**：极致留白，中心放置 iML Agent 品牌 Logo。
- **文案**：
    - 主标题：“人人皆可定义。”
    - 副标题：“感知、思考、执行。iML Agent 让每一台电脑都进化为拥有真实意志的桌面级智能体。”

### 模块 B：Solution - 解决什么问题？
- **对比设计**：左侧展示传统的“哑巴对话”，右侧展示 iML Agent 的“全能执行”。
- **痛点点睛**：AI 终于有了“手脚”。

### 模块 C：Execution Loop - 深度交互三要素
1.  **跨端感知 (Omni-Perception)**：监听文件系统变动、识别活跃应用上下文、读取系统通知及剪切板。
2.  **多维执行 (Multi-Execution)**：从 Shell 脚本到 GUI 模拟，从浏览器自动化到原生应用 API 调用，无缝衔接。
3.  **认知持久 (Vectra Persistence)**：将操作系统中流转的所有碎片化信息，转化为可检索的本地向量知识库。

### 模块 D：Showcase - 跨软件联动案例 (Storyline)
- **场景演示**：一个流畅的联动序列动画：
    - 传感器检测到“下载文件夹”有新 PDF -> AI 自动解析 -> 在“终端”提取数据 -> 启动“浏览器”搜索背景 -> 最终在“本地 Canvas”合成周报并推送到钉钉/Slack。
    - 视觉重点：不同软件图标之间的连线与数据流转动画。

### 模块 E：Evolution - 无限技能扩展
- **核心卖点**：除了内置的 20+ 企业级技能，iML Agent 支持**“双重进化”**。
- **能力 1：格式化生成技能**：用户输入自然语言描述（如：“帮我写一个能自动压缩所有 1GB 以上图片并上传到 OSS 的技能”），系统生成标准、可用的技能插件 (Standard Skill)。
- **能力 2：一键生成工作流**：用户通过交互对话（如：“帮我分析竞品并整理成周报”），系统自动编排并生成标准工作流 (Standard Workflow)。
- **视觉**：采用双循环动效演示。
    1.  **Skill Loop**：文字 -> 代码块 -> 紫色技能卡 -> 存入矩阵。
    2.  **Workflow Loop**：对话气泡 -> 流程图 -> 蓝色工作流卡 -> 存入矩阵。

### 模块 F：Privacy - 隐私第一
- **强调**：Local First, Privacy Always. 数据不出本地。

## 5. 技术方案
- **框架**：Vanilla HTML/CSS + JS (GSAP 为辅助)。
- **动效库**：gsap (用于精准控制执行流动画)。
- **字体**：Inter (正文) + Outfit (标题)。

## 6. 预期效果
用户进入页面 3 秒内理解“这是一款能干重活的桌面级 AI”，并产生强烈的“这是我从未见过的专业级工具”的认同感。
